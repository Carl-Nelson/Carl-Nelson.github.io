Part 2: Questions B

  1. They're... completely different languages, though I'm mostly familiar with Java. Java is compiled to "bytecode" which the JVM converts on the fly into executable code.
    Javascript is interpreted, run directly by the interpreter(like an internet browser perhaps) without previous compilation. Which makes it somehow even slower than Java!
    It's also even more easily portable than Java, allows dynamic typing (some people view that as a positive), and the web as we know it wouldn't function without it.
  2. '+' is a binary operator. It adds the second operand to the first one, with the precise behavior depending on the type(s) of data. Actually apparently you can have nothing at all on the left. That's weird.
    I guess ' +whatever' has the effect of casting whatever into a number.
    '++' is a unary opereator. It just increments the operand by one.
  3. window.alert outputs text to a popup window. console.log outputs text to the console, which you can only see if you've got the console open.
    document.write writes html(!!!) to the page. Tags n' everything.
  4. '=' is an assignment operator. It assigns the value of the second operand to the first, which should be a variable. '==' is a comparison operator. It returns true if the two operands are equal, false if not. It has some weird as heck behavior in javascript, due to the funny typing.
    '===' returns whether the two operands are equal AND the same type.
  5. While checks the loop condition before entering the body of the loop, do while checks it afterwards. Do while will always run at least once; a while loop might never.
